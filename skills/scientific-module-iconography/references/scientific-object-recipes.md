# Scientific Object Recipes

## External Census Summaries

Use:

- 3 to 5 compact summary tiles
- mini histograms or structured summary cards
- unified family, not random icons

Avoid:

- oversized chart panels
- dashboard-like UI widgets

## PUMS Microdata

Use:

- record card
- stacked row sheet
- small table slab

Visual cue:

- lighter than the model stage
- clearly train-only, not a processing module

## Joint heatmap or probability table

Use:

- translucent matrix slab
- shallow tilt
- crisp grid
- mild depth

Avoid:

- flat spreadsheet look
- giant rainbow grid

## Population output

Use:

- grouped stylized person tokens
- clean geometric silhouettes
- 5 to 8 figures
- same family, different colors

Avoid:

- emoji people
- childish clipart

## When objects should look quieter

Objects in Stage 1 and Stage 3 should be visually calmer than the Stage 2 modules. If the inputs or outputs are louder than the model-learning stage, the visual hierarchy collapses.
